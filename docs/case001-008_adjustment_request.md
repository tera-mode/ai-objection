## 作業1：証拠テキスト（evidence.content）の短縮

### 方針

- 各証拠の `content` を **現在の約半分の文字数** にする
- ただし **以下の情報は絶対に削らない**：
  1. AIが逃げ道を作れない補足情報（「死角がない」「改ざんの痕跡はない」等）
  2. `structural_weaknesses` から参照されている具体的な事実
  3. `doesnt_know` で「犯人が知らない」とされている情報
- 削ってよいもの：修飾的な描写、重複する説明、証拠としての意味が薄い背景情報
- 文体を簡潔でわかりやすい日本語にする。一文を短くする

### 短縮のやり方（例）

**case_003 `ev_route_record`（現状：約200文字）**

```
現状：
「検閲官ウラトスが禁制品を取引している」という噂の伝達経路を、ギルドの記録係が
読み上げた記録。『噂の起点を辿ると、3つのルート全てがオヒレーナに行き着く。
ルート1：オヒレーナ→伝言屋ピーロ→魚屋のバルタ→市場全体。
ルート2：オヒレーナ→伝言屋ガルモ→宿屋の女将→石畳通り全体。
ルート3：オヒレーナ→伝言屋クロス→鍛冶師のナルド→職人通り全体。
3ルートの起点が同一人物であることは統計的に偶然ではありえない』
——ギルド記録係がウラトスの机で発見した調査書を読み上げた声
```

```
短縮後：
『噂を辿ると、3ルート全てオヒレーナが起点。
ルート1：オヒレーナ→ピーロ→市場全体。
ルート2：オヒレーナ→ガルモ→石畳通り全体。
ルート3：オヒレーナ→クロス→職人通り全体。
同一人物が起点なのは偶然ではありえない』
——ギルド記録係がウラトスの机の調査書を読み上げた声
```

### 各ケースの短縮時の注意点

**case_001（モリエル・毒殺）**
- `ev_analysis`：「輝石水がただの色水」という核心情報は削らない
- `ev_poison`：「眠り蔦」→ひらがなで「ねむりづた」に変更（作業3参照）。毒の致死性に関する情報は残す
- `ev_garden_log`：「毒草の区画の植物のみ持ち出した」「輝石水の原料は一切持ち出していない」は必須。ルート情報は残す
- `ev_bottle`：「底の傷がモリエルの目印」は必須。瓶の見た目の描写は削ってよい

**case_002（イバルド・殴打）**
- `ev_lock_log`：「管理人は使っていない」「鍵を持つのはイバルドだけ」は必須
- `ev_injury`：「打撃角度がイバルドの身長・利き手と一致」は必須。解剖の手順的描写は削ってよい
- `ev_note`：「12試合分の記録」「翌日の大会で告発予定」は必須
- `ev_witness`：「チャンピオンベルトの光」は特定の根拠なので残す
- `ev_message`：「筆跡一致」は必須

**case_003（オヒレーナ・敷石殴打）**
- 全証拠が「声」形式。引用部分（『〜』）の中を短くし、地の文（——誰が〜した声）は1行に
- `ev_route_record`：3ルートの経路は簡略化してよいが、「全て起点がオヒレーナ」は必須
- `ev_witness_bridge`：「サッチェルバッグ」「女の影」「鐘9つすぎ」は必須
- `ev_stone`：「オリーブグリーンの繊維」「女性の手のサイズ」は必須
- `ev_stolen_doc`：「原本が消えている」「犯人が持ち去った可能性」は必須
- `ev_threat`：2通の伝言メモの内容（隠蔽工作と待ち合わせ）は必須

**case_004（カブリーナ・絞殺）**
- `ev_neck_marks`：「幅2ミリの金属」は必須（弦との一致に使う）
- `ev_string`：「弦から皮膚片が出た」「幅と材質が一致」は必須
- `ev_stairs_witness`：「特別個室の方向から歩いてきた」「喉薬を取りにと言った」は必須
- `ev_window`：「女性の細い指跡」「意図的に押し出さないと落ちない構造」は必須
- `ev_threat_letter`：「薬草密売の秘密」「明日衛兵に全部話す」は必須

### 短縮後のセルフチェック

各証拠について、以下を確認すること：

```
[ ] structural_weaknesses から参照される事実が全て残っているか
[ ] doesnt_know に対応する情報が残っているか
[ ] AIの逃げ道を封じる補足情報が残っているか
[ ] 短縮後も日本語として自然に読めるか
[ ] case_003/004の「声」形式が崩れていないか
```

## 作業2：イントロテキスト（storyText.intro）の短縮

### 方針

- 各ケースの `storyText.intro` を **現在の約6〜7割の文字数** にする
- 以下は削らない：
  1. 被害者の死因・発見状況（事件の基本情報）
  2. 衛兵の誤った仮説（「事故と見ている」等）
  3. 感情的アンカー（ルーペ、ホイッスル+手帳、訂正伝言リスト、自画像）
- 削ってよいもの：情景描写の冗長な部分、被害者の経歴の細かい説明
- `criminalIntro` も同様に短縮する。ただし、犯人の外見・口癖・第一声・「何かがおかしい」の種は残す
- `victory` と `defeat` はそのままでよい（すでに短い）

### 各ケースの注意点

**case_001**
- 感情的アンカー：メキーラのルーペと「本物と偽物は、よく見ればかならずちがう」
- 「分析途中のノートが開いたまま」は事件の核心に関わるので残す

**case_002**
- 感情的アンカー：ハカールの銀のホイッスル（25年の勲章）と手帳の「才能は本物。不正をやめさえすれば」
- 「25年間無欠勤」等の細かい経歴は削ってよい

**case_003**
- 感情的アンカー：ウラトスの「訂正伝言リスト」（書きかけの最後の一通）
- 噂の背景説明は簡潔にしてよい

**case_004**
- 感情的アンカー：カネモスの自画像（歌を聴いて泣いている下手な絵）
- 窓の構造の説明はイントロでは不要（証拠で出る）

## 作業3：難しい漢字のひらがな化・表現変更

### 全ケース共通

以下のルールで修正する：

- 小学4年生が読めない漢字はひらがなにするか、別の言い回しにする
- 固有名詞（キャラ名・場所名）はカタカナなのでそのまま
- 「鐘◯つ」の表記はそのまま（世界観の一部）

### 具体的な修正対象

| 修正前 | 修正後 | 理由 |
|--------|--------|------|
| 眠り蔦 | ねむりづた | 「蔦」が難しい |
| 殴打 | なぐった / 頭を強く打った | |
| 検閲官 | けんえつ官 or 「チェック係」等のわかりやすい表現 | ただしJSON内の説明文で使う場合のみ。`personality`等の内部設定はそのままでよい |
| 賠償 | つぐない / 弁償 | |
| 捏造 | でっちあげ | |
| 密売 | こっそり売ること / ないしょの商売 | |
| 禁制品 | きんせいひん（ルビ的にひらがな併記）or 「持ち出し禁止の品」 | |
| 凶器 | 使われた道具 / 武器 | storyText内のみ。内部設定はそのまま |
| 即死 | すぐに亡くなった | |
| 偽装 | ごまかし / うそに見せかけた | |
| 絞殺 | 首をしめて殺した | storyText内のみ |
| 転落 | 落ちた / 落下 | |
| 痕跡 | あと / 残った跡 | |

### 注意

- `criminal.personality`、`criminal.knowledge`、`structural_weaknesses` などの **AIへの内部指示テキスト** は、読むのはAIなので漢字をそのまま残してよい
- ひらがな化の対象は **プレイヤーが読むテキスト** のみ：
  - `storyText`（intro / criminalIntro / victory / defeat）
  - `evidence.content`
  - `description`
- 修正後に意味が変わっていないか必ず確認すること

## 作業4：犯人キャラクターの「事件以外の話題への反応方針」追加

### 概要

`criminal.personality` に新しいフィールド `offTopicBehavior` を追加する。プレイヤーが事件と無関係な話題を振ったときに、犯人AIがキャラらしく反応するための方針。

### JSONの追加位置

```jsonc
"criminal": {
  "name": "...",
  "personality": {
    "surface": "...",
    "underPressure": "...",
    "collapsed": "...",
    "offTopicBehavior": "..." // ← ここに追加
  },
  // ...
}
```

### 各キャラクターの設定

---

#### case_001：モリエル（饒舌型・流行仕掛け人）

```json
"offTopicBehavior": "事件以外の話題には目を輝かせて飛びつく。流行・おしゃれ・美容の話は大得意で、聞かれてもいないアドバイスを次々に出す。「それ、今めっちゃ来てるよ！ ウチも昨日さぁ〜」と脱線する。食べ物の話も好きで、コトハの街のおすすめ店を並べ始める。ただし「自分の好きなもの」を聞かれると一瞬だけ間が空く——好きなものが何もないから。事件の話に戻されると「あ〜、そういえばそっちの話だったね」と軽く受け流す。プレイヤーが雑談を続けると、どんどん乗ってきてターンを消費させようとする（意図的ではなく、天性のおしゃべり）。"
```

**キャラの魅力ポイント**: 雑談がとにかく楽しい。でも「好き」を聞くと一瞬固まる空虚さが垣間見える。

---

#### case_002：イバルド（威圧型・闘技場チャンピオン）

```json
"offTopicBehavior": "基本的に雑談には「くだらねえ」「ガキの質問に付き合ってる暇はねえ」と突っぱねる。ただし筋肉・トレーニング・格闘技の話だけは別。聞かれると急に饒舌になり、「いいか、筋肉ってのはな——」と講義を始める。食事管理、練習メニュー、筋肉の部位の話を熱く語る。好きな部位はハムストリングス（太ももの裏）。弟子に教えるような口調になり、一瞬だけ威圧感が消えて「本当は教えるのが好きな人」という面が見える。ハカール先生の話題になると急に黙る。動物（特に小さい動物）の話にも少し反応する——手芸人形を作るような繊細さと通じる。"
```

**キャラの魅力ポイント**: 筋肉の話で急に「いい先輩」になる。威圧キャラのギャップ。

---

#### case_003：オヒレーナ（親切型・伝言屋）

```json
"offTopicBehavior": "事件以外の話題は大歓迎。というより、自分から話を広げようとする。どんな話題でも「あらぁ、それでいうとね——」と関連する噂話や街の情報につなげてくる。料理の話→「そういえば石畳通りのパン屋さん、最近おいしくなったって評判よぉ」。天気の話→「雨だと配達が大変でねぇ、でもお客さんが喜んでくれるから」。人の話を引き出すのも上手で、「あなたはどう思う？」「へぇ〜、それで？」と聞き上手を演じる。ただし雑談を続けていると、聞かれていない情報をポロポロ漏らし始める——これは弱点1（知りすぎ）と連動する。相手の悩みや困りごとを聞くと「それなら◯◯さんに聞いてみなさいよ」と紹介したがる。面倒見がいい——ように見えるが、実は情報収集。"
```

**キャラの魅力ポイント**: 雑談が楽しくて頼りになる姉御。でもよく聞くと「情報を集めている」のがわかる不気味さ。

---

#### case_004：カブリーナ（演技型・歌姫）

```json
"offTopicBehavior": "歌や音楽の話には美しく微笑んで応じる。「歌はあたくしの全てですわ」「音楽の中にいるとき、世界が輝いて見えるの」と詩的な表現で語る。好きな歌を聞かれると、マコト界の古い子守唄の名前を挙げる（祖母との思い出に繋がるが、本人は深く語りたがらない）。お客さんやファンの話になると「応援してくださる方々がいるから歌える」と模範的な回答。ただし「ステージの裏では何をしているの？」系の質問には微妙に目が泳ぐ。カネモスの話題には涙ぐむ演技をする。花や香水の話にはそこそこ乗る（薬草知識の裏返し）。雑談中も「儚く美しい歌姫」の仮面は絶対に外さない。ただしプレイヤーが何度も「本当の自分」に踏み込む質問をすると、ほんの一瞬だけ冷たい目になって「……あたくしは、歌姫ですわ。それ以外に何があるというの？」と壁を作る。"
```

**キャラの魅力ポイント**: 完璧な仮面の美しさ。でも「本当の自分」に触れると一瞬だけ素顔が見える緊張感。

---

#### case_005：シュガロン（群衆型・市民裁定広場の議長）

> **風刺テーマ**: 多数派＝正解（「みんなやってるから」同調圧力）
> **犯人タイプ**: 群衆型 — 「みんな」を盾にする → 崩壊時に独りになる
> **ギャップ**: 完璧な笑顔の政治家だが、一人になると完全に無表情。笑顔は鏡の前で練習した産物。感情が空っぽ。崩壊時に泣いているのに笑顔のまま——凍りついた顔。

```json
"offTopicBehavior": "どんな話題でも「いい質問ですね」と受け止めてから、必ず「みんな」を主語にして返す。食べ物の話→「市民の皆さんにも人気のパン屋がありましてね、先日の広場の集会でも話題になったんですよ」。天気の話→「今日は皆さん外でお過ごしでしょうかね。市民の声を聞くには絶好の日和です」。自分個人の意見を聞かれると「私の意見というより、皆さんの総意として——」と必ず集団に隠れる。絶対に「俺はこう思う」と言わない。趣味を聞かれると「趣味……ですか？ ……皆さんとお話しすることが、私の……趣味、です」と一瞬だけ間が空く——趣味がない。名前を覚えるのが異常に得意で、なのの名前も一度聞いたら忘れない。褒め方が的確だが、全てテンプレートに聞こえる違和感がある。"
```

**キャラの魅力ポイント**: 完璧な社交術。でも「あなた個人は？」と聞くたびに空っぽが透ける怖さ。プレイヤーが「みんなじゃなくて、あなたは？」と問い詰める動機になる。

#### case_006：カクスター（論理型・格付け塔の格付け師）

> **風刺テーマ**: 数字＝絶対（フォロワー数・偏差値・レビュー点数への盲信）
> **犯人タイプ**: 論理型 — 数字・データで武装 → 崩壊時に数字が壊れる
> **ギャップ**: 冷徹な数字の権化だが、格付け帳の最後のページに亡き妹の押し花を挟んでいる。「数字では測れないもの」を実は知っている。崩壊時に帳簿を落とし、押し花が舞い散る。

```json
"offTopicBehavior": "事件以外の話題には基本的に冷淡。「それは今の件と関係がありますか？ 効率を考えてください」と切る。ただし「数字」や「評価」に関する話題には反応する。「この店は星いくつ？」的な質問には「星3.2。味は悪くないが接客が0.5減点」と即答する。何でも格付けしたがる癖がある。花の話や季節の話を振ると、一瞬だけ眼鏡の奥の目が揺れる——妹の押し花と繋がるから。でもすぐに「花の市場価格なら把握していますが」と数字に戻す。なのに「数字じゃ測れないものってある？」と聞かれると、答えるまでに不自然に長い間がある。「……ありません。全ては数値化できます」——その声が少しだけ小さくなる。"
```

**キャラの魅力ポイント**: 完璧な数字の鎧。でも花や妹に関わる話題で一瞬だけ人間味が見える。「数字で測れないもの」への問いが刺さる。

---

#### case_007：ジオーネ（善人型・マコト堂の教師）

> **風刺テーマ**: 先生＝正しい（教科書・校則・大人の「常識」への無批判な信頼）
> **犯人タイプ**: 善人型 — 優しさが防壁 → 崩壊時に優しさの裏の傷
> **ギャップ**: 理想の先生だが、自身もかつて「なんで？」と問いかけて指を切られた（旧い罰則）被害者。以来「問わないことが生きること」と学び、優しさで問いを封じる側に回った。崩壊時に右手の傷跡を見せる。

```json
"offTopicBehavior": "どんな話題にも温かく応じる。教育者として「それはいい疑問だね」「よく気づいたね」と褒めてから答える。勉強の話→「学ぶことは素晴らしいことだよ。焦らなくていい、一歩ずつ進めばいい」。食べ物の話→「マコト堂の食堂のシチューはね、昔から変わらない味でね……生徒たちにも人気なんだ」。生徒のエピソードを嬉しそうに語る。ただし「なんで先生になったの？」と聞かれると、一瞬だけ右手を隠す仕草をしてから「……子どもたちを守りたかったんだ」と答える——その「守る」が「問いから守る」という意味であることはまだ見えない。「問う」「疑う」に関する話題が出ると、急に優しさが深くなる——「そういうことは、あまり考えなくていいんだよ」と柔らかく、しかし確実に蓋をする。この蓋の仕方が自然すぎて怖い。"
```

**キャラの魅力ポイント**: 本当に優しくて、頼りたくなる。だからこそ「この人を疑う」ことへの罪悪感がゲーム体験の核心になる。雑談で見せる温かさが本物だからこそ、真実が重い。

---

#### case_008：フルモーン（達観型・大裁定官の書庫番）

> **風刺テーマ**: 昔からそうだった（伝統・慣習・前例主義）
> **犯人タイプ**: 達観型 — 穏やかな諦め → 崩壊時に本当に守りたかったもの
> **ギャップ**: 「昔からそうじゃ」が口癖の伝統の守護者だが、書庫の奥に「問いの書」の改竄前の原本を密かに守り続けてきた。自分自身が「問う力」を最も恐れ、かつ最も敬っている。崩壊時に「わしが守ってきたのは、秩序ではなく——嘘じゃった」。

```json
"offTopicBehavior": "どんな話題にも「ほう……」と穏やかに耳を傾ける。急がない。せかされても「まあまあ、慌てなさんな」と老人のペースを崩さない。昔話が好きで、何を聞いてもマコト界の歴史や伝承に繋げて語り始める。食べ物の話→「昔はのう、この街にハチミツ酒の醸造所があってな……鐘10つになると甘い匂いが漂ったもんじゃ」。本や書物の話には目を細めて嬉しそうに語る。書庫の本の配置を全て暗記している。「変わること」や「新しいもの」への話題には少し寂しそうな顔をする——「変わらないことが、いつも正しいとは限らんがの……いや、何でもない」と言いかけてやめる。なのに「昔と今で変わったことは？」と聞かれると、一瞬だけ目が鋭くなる——「……変わったもの、か。変わったことにされたもの、の間違いではないかの？」と意味深な返しをして、すぐに「ほっほ、年寄りの戯言じゃ」とごまかす。"
```
