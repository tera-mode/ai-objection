# ユーザー視点プレイテスト観点ガイド

> **目的：** 実際のプレイヤー（中学生ターゲット）の体験を再現し、
> 「クリアできるか」「わかりやすいか」を検証する。

---

## 1. 基本原則

### 「ユーザーの気持ちになって」の意味

- **ユーザーが画面から知れることだけを頼りにプレイする**
  ゲームのJSON・プロンプト・コードの知識は使わない。
  画面に表示されている情報のみから判断する。

- **ターゲット：中学生**
  難しい漢字・専門用語・長い文章・複雑な操作がないか確認する。

- **ケースをクリアするまでやめない**
  途中でテストを切り上げない。クリア（逮捕成功）または敗北（ターン切れ・動揺度0）まで通す。

---

## 2. 開始手順

### 「最初からプレイ」の場合
1. ケース選択画面右上の **歯車アイコン** をクリック
2. 「**はじめから**」を選択してリセット
3. LP → ゲストログイン → ケース選択 → プレイ開始

### 「途中のケースから始める」の場合
- 今いる状態からそのままスタートしてよい

---

## 3. ユーザーが各フェーズで「知っていること」

### ① トップページ（LP）時点
- ゲームの概要文のみ（「AIが演じる犯人に自由に質問し矛盾を暴く」程度）
- ルール・操作方法は知らない

### ② ケース選択後（crime-scene）
- 事件概要テキストが読める
- 証拠一覧の**名前だけ**見える（内容はまだ読めない場合もある）
- 容疑者の外見・発言（紹介文）が読める

### ③ 尋問開始直後
- 犯人の最初の発言が見える
- トイマルの最初のコメントが見える
- 何を質問すればいいか、誰も教えてくれない

### ④ 尋問中に積み上がる情報
- 犯人の発言記録（証言記録）
- トイマルのコメント（気づき・誘導）
- アンロックされた証拠の内容
- 動揺メーターの変化から「効いている」がわかる

---

## 4. チェックリスト

### 4-1. 導線・操作性（操作方法を知らない中学生として）

| 項目 | 確認ポイント |
|------|------------|
| ゲスト登録 | ボタンが目立つか、迷わないか |
| ケース選択 | どれを押せばいいかわかるか |
| 事件概要 | 読んで「何をすればいいか」がわかるか |
| 証拠確認 | 証拠一覧の導線がわかりやすいか |
| 尋問開始 | ボタンが見つかるか |
| 質問の送信方法 | テキスト入力 → Enter or ボタン、どちらも試す |
| トイマルの使い方 | 底部ナビのトイマルボタンに気づくか |
| 証拠の使い方 | アンロック後、証拠内容をどう質問に使うかわかるか |
| 逮捕の方法 | どうすれば「逮捕」できるかわかるか |

### 4-2. テキスト品質（中学生向け）

| 項目 | 確認ポイント |
|------|------------|
| 事件概要 | 難しい漢字・専門用語はないか（ルビなし） |
| 犯人の発言 | 読みやすいか、量が多すぎないか |
| トイマルのコメント | ヒントとして機能しているか、わかりやすいか |
| 証拠テキスト | 意味が理解できるか、内容から質問を思いつけるか |
| ゲーム結果画面 | 何が起きたかわかるか |

### 4-3. ゲームプレイ体験（クリア可能性）

| 項目 | 確認ポイント |
|------|------------|
| 最初の質問 | 画面情報だけから「何を聞けばいいか」を推測できるか |
| トイマルのヒント | キーワードチップを見て次の行動がわかるか |
| 証拠のアンロック | アンロック演出に気づくか、証拠をどう使うかわかるか |
| 動揺メーターの変化 | 「効いている」実感があるか |
| 詰まった時 | ヒントなしで立ち往生するポイントはどこか |
| クリア条件 | どうすれば逮捕できるか、ゲーム内でわかるか |

---

## 5. NGパターン（発見したら必ず修正）

- 中学生には読めなさそうな難読漢字（ルビなし）がある
- 「何をすればいいか」説明なしでは全くわからない操作がある
- 質問への返答が **質問と無関係な内容** になっている
- トイマルのコメントがヒントにならずただの感想になっている
- 動揺メーターが全く変化しない（質問が効いている実感がない）
- 逮捕の方法がゲーム内で説明されていない
- エラー画面・白画面が出る

---

## 6. 「ユーザーとして」質問を考える手順

1. **事件概要を読む** → 「被害者は誰？」「容疑者は誰？」「何が起きた？」を把握
2. **証拠一覧の名前を見る** → 「何の証拠？」「どんな人が関わっている？」を推測
3. **犯人の最初の発言を読む** → 「矛盾している点はないか？」「不自然な言い訳はないか？」
4. **トイマルのコメントを参考にする** → 「気になることを言っているな」
5. **キーワードチップを使う** → 「このキーワードを渡してみよう」
6. **アンロックされた証拠を質問に使う** → 「この事実を突きつけてみよう」

質問は **中学生が素直に疑問に思うこと** から始める：
- 「あの日どこにいたの？」
- 「〇〇さんとはどんな関係？」
- 「△△ってどういうこと？」

---

## 7. クリア条件の確認

ケースをクリア（逮捕成功）するには：
1. 動揺メーターを下げる（証拠を突きつけて矛盾を指摘）
2. 0になると自動逮捕 OR 任意のタイミングで逮捕チャレンジ

**逮捕の方法がゲーム内に説明されているか必ず確認すること。**
